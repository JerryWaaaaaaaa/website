### Design changes
- Two global motion easings added to `:root` — `--ease-out: cubic-bezier(0.23, 1, 0.32, 1)` for snappy UI feedback and `--ease-emphasized: cubic-bezier(0.16, 1, 0.3, 1)` for longer-travel entrances. Hard-coded cubic-bezier values throughout existing code replaced with these tokens.
- BuiltDifferentiate titles for steps 2/3/4 now carry inline decorations: step 2 gets a `TitleBadge` pill ("missing from most AI slide tools") + an `Underline` draw-on beneath "Fully editable"; step 3 gets a `Highlight` marker sweep + `AISparkle` gradient emblem on "Inline AI"; step 4 gets an inline `ScreenshareChip` (dark pill with monitor icon) + `PointingHand` cursor overlay. All decorations use `em`-relative sizing and `--text-highlight` / `--bg-accent-light` / `#d4e6fc` tokens.
- Crossfade transitions across the board replaced with a layered z-index stacking model: incoming slot fades in on top with a `blur(4px) → blur(0)` resolve, outgoing slot held at full opacity beneath. This eliminates the luminance dip (dark backing bleed-through) during mid-swap.
- New shared reveal-on-enter recipe: text slides up 16 px + fades (`450ms --ease-emphasized`); media scales from 0.96 + fades (`550ms --ease-emphasized`). Stagger via inline `--reveal-delay`. Entrance fires once per element via `data-enter` attribute.
- Decoration entrance sequencing after reveal: badge drifts up (120 ms delay) → underline scaleX draw-on (250 ms) → highlight sweep (280 ms) → sparkle pop (440 ms) → hand tap keyframe (360 ms).
- Use Case tabs redesigned: tabs no longer flex-grow (the first tab no longer stretches); all tabs are natural-width with 48 px gap. New variant B adds a sliding `--bg-accent-light` fill element + SVG curved "shoulder" flares that transition `left`/`width` at `360ms --ease-emphasized`.
- Persona panel media now uses stacked-image crossfade with outgoing blur (`filter: blur(6px) → blur(0)`, `260ms --ease-out`) instead of keyed remount, preventing empty-frame flicker.
- All motion work includes full `prefers-reduced-motion: reduce` fallbacks that land decorations in final state and drop travel/scale/blur.

### Integration notes for Curry
- **No new npm dependencies.** All decoration pieces are inline SVG/CSS — no icon library or animation library to install. The `AISparkle` gradient uses `id="kp-sparkle-grad"` which must stay unique in the DOM (or be converted to a CSS gradient if multiple instances coexist on the same page).
- **The V5D scroll paint is intentionally imperative** (DOM refs, not React state) for 60 fps scroll-linked animation. If you move to a framework with a virtual list or scroll library, the `paint()` function and its ref-writing pattern are the core to preserve or replace; the React state (`activeIndex`) is only used for the discrete active-step UI.
- **`data-enter` is the universal reveal trigger.** All reveal + decoration entrances key off `[data-enter='true']` in CSS. Production can swap the IntersectionObserver-based `useRevealOnce` for any scroll library (e.g. Framer Motion's `whileInView`) as long as it sets that attribute or toggles the equivalent class. The stagger timing lives in inline `--reveal-delay` values, not in JS.
- **The old `UseCaseV5` is gone.** The page now renders `UseCaseV5B` (sliding-fill tabs). The shared `PERSONAS` data and `PanelCopy`/`PanelMedia` components in `shared.tsx` are ready to support additional tab-transition variants (e.g. a Variant A crossfade) by importing the same shared module.
