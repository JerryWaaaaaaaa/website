### Integration notes for Curry
- **Mockup floats are parent-owned, not encapsulated.** If you refactor a mockup into a self-contained component, note the floating widgets (comments/tooltips/etc.) are deliberately rendered by `ProductSuiteV5` outside the editor card so they can visually spill past its edges. Keep the mockup ↔ float-widget split, and keep the `active` prop driving the intro.
- **The mockups do NOT use DESIGN.md tokens by design.** They hard-code Inter/SF Pro + Figma hexes per the design brief. Don't migrate them onto the site type/color scale.
- **`V5Tuning` is a prototype-only dev harness** — it's a live dial/URL-param tuning panel used to place floats and set timings. In production, freeze the resolved values as static CSS custom props (`--float-*`, `--paper-*`, `--sheet-*`, `--psuite-v5c-cycle`) and drop the panel + `dialkit`/URL-sync plumbing. The current defaults in `V5Tuning.tsx` are the intended production values.
- **Uncommitted large video assets exist** (`canvas-website.mp4`, `Sheets- finalized version.mp4` in `public/product-suite-assets/`, untracked). The live mockups replace these videos, so they're likely dead weight — confirm with Jerry before shipping either the videos or the now-unused video code path (`product.video` branch still present in ProductSuiteV5).
- **No new npm dependencies** — everything is React + CSS + existing `dialkit` tuning util. Mockups self-scale via `ResizeObserver` against fixed design boxes; all intro animations are CSS-driven off a single `data-enter` gate with per-element `--d` delays, so verify `prefers-reduced-motion` behavior when productionizing.
