### Integration notes for Curry
- **New dependency:** run `npm install` — this push adds `dialkit ^1.3.0` (powers both the variant picker and the design-tuning sliders). `dialkit` and the entire `V5Tuning` / `dialUrlSync` / `?tune=` / `?<group>=variant` URL machinery are **prototype authoring tools, not production** — in prod, bake the chosen variants and tuned CSS-token values in as static defaults and drop dialkit + the dial UI entirely.
- **Product videos are no longer in git** (`public/product-videos/` is gitignored; the old `.mp4`s were deleted). Slides no longer needs a video (it's DOM-rendered), but Canvas/Sheets still reference `/product-videos/*.mp4` in `ProductSuiteV5.tsx` — Curry must source those assets out-of-band or the non-Slides tabs show only the poster image.
- **`SlidesMockup` hardcodes a 1280×694 design canvas** and scales by width; the card's `aspect-ratio` in `ProductSuiteV5.css` must stay `1280/694` or the slide padding breaks. Slide artwork lives at `/public/slides-mockup/slide-0{1..8}.jpg` (large, ~3–4MB each — compress/convert to WebP before prod; a `png2webp` script exists).
- **`StepMedia` swaps the animation in by matching the literal image path `/core-features/meeting-deliverable.png`** because `keyPoints.tsx` can't be edited in this prototype. In production, model this as a first-class `media.kind` (e.g. `layers`/`animation`) on the key-point data instead of a path string match.
